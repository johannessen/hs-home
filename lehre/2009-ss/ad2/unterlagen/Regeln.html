Regeln

Die folgenden Stilregeln müssen für im Rahmen dieser Veranstaltung geschriebenen Code beachtet werden:

  • immer im Kopf jeder Datei Autor (voller Name) und Datum (wahlweise erste Erstellung oder letzte Bearbeitung) nennen
  • Groß-/Kleinschreibung von Bezeichnern: Klassen immer CamelCaseNamesAbc, alles andere immer middleCaseNamesAbc (Ausnahme: Konstanten, Packages)
  • keine Unterstriche verwenden (Ausnahme: Konstanten)
  • Bytewerte außerhalb des Bereichs von 32 bis 126 (0x20 bis 0x7e, einschl.) sind außerhalb von Kommentaren nur erlaubt g. d. w. ihre Kodierung in jeder Datei einheitlich ist und im Kopf angegeben wird. (Diese Regel ist z. B. dann erfüllt, wenn in der Datei ausschließlich druckbare ASCII-Zeichen vorkommen.)
  • optionale Blöcke immer explizit verwenden (z. B. if (true) { doSomething(); } )
  • nirgendwo wird public, private oder protected verwendet
    (Ausnahme: public static void main (String[]) oder andere Fälle, wo es unbedingt notwendig ist)
  • Typen nicht auseinanderreißen (immer String[] args statt String args[] schreiben)

Diese Regeln gelten zwar nicht für die Prüfung, können jedoch auch dort hilfreich sein.
